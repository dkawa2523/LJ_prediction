# Coding Policy（コーディング規約）

このドキュメントは実装品質と保守性を保つための規約です。

## 1. 基本
- Python 3.10+（推奨）
- type hints を可能な範囲で付ける（特に public API）
- 公開関数/クラスには docstring を付ける

## 2. エラーとログ
- ユーザー入力由来のエラーは「何を直せば良いか」をメッセージに含める
- 例外の握りつぶしは禁止（必要なら `raise ... from e` を使う）
- ログは `logging` を使い、print 乱立を避ける

## 3. 依存追加のルール
- 新しい外部依存は最小限に
- 追加理由・代替案を `work/tasks/*.md` に明記
- secrets/APIキーはコードに埋め込まない（`docs/08` 参照）

## 4. パフォーマンス
- 特徴量計算はキャッシュ戦略を検討（再計算の無駄を減らす）
- 大規模化に備え、データローダはストリーミング/バッチを意識する

## 5. 互換性
- config key / artifact 形式の破壊は禁止（破るなら ADR）
